# 板块三详解：向量——点积与叉积

## 3.1 先理清：什么是向量？

你已经会在 Python 中写 `np.array([x, y, z])`。但这只是向量的**数据表示**。向量本身到底是什么？

**向量的两种视角**：

| 视角 | 理解 | 例子 |
|------|------|------|
| **几何** | 空间中一个有方向、有长度的箭头 | 从原点指向点 `(2, 3, 5)` |
| **代数** | 一组有序数 `(x, y, z)` | `np.array([2, 3, 5])` |

在 3DGS 中，每个高斯基元的**位置 `μ`** 就是一个 3D 向量：
```python
mu = np.array([1.5, 0.3, 4.2])   # 椭球中心在空间中的位置
```

---

## 3.2 向量的长度（模）

```python
v = np.array([3, 4])
norm = np.linalg.norm(v)    # = 5.0
```

公式就是勾股定理：`|v| = √(3² + 4²) = √(9+16) = 5`

**单位向量**：长度为 1 的向量，只保留方向信息：

```python
v = np.array([3, 4])
v_hat = v / np.linalg.norm(v)
# v_hat = [0.6, 0.8]
# 验证：0.6² + 0.8² = 0.36 + 0.64 = 1.0 ✓
```

**为什么需要单位向量？** 当你想"往某个方向走"，你需要一个纯方向（长度 1），然后乘以步长。3DGS 中视线方向就是单位向量——只关心往哪看，不关心看多远。

---

## 3.3 点积 —— 两个向量的"对齐程度"

### 两个定义（数学上等价）

```
代数定义：a·b = a₁b₁ + a₂b₂ + a₃b₃      ← 代码里实际算的
几何定义：a·b = |a|·|b|·cos(θ)         ← 用来理解的
```

### 等价性证明（这是理解点积的关键）

**已知**：|a|, |b|, 夹角 θ

**用余弦定理**：
```
|c|² = |a|² + |b|² - 2|a||b|·cos(θ)     ← c = a - b
```

**用代数定义展开 |c|²**：
```
|c|² = (a-b)·(a-b)        ← 代数的点积
     = a·a - a·b - b·a + b·b
     = |a|² - 2(a·b) + |b|²
```

**对比两个表达式**：
```
|a|² + |b|² - 2|a||b|cos(θ)  =  |a|² - 2(a·b) + |b|²
                        -2|a||b|cos(θ)  =  -2(a·b)
                           |a||b|cos(θ)  =  a·b   ✓
```

**这个证明告诉你**：点积的两种定义不是巧合，是余弦定理的必然推论。

### 点积告诉你什么？

```python
a = np.array([3, 0])       # 指向正右方
b1 = np.array([2, 1])      # 右上，夹角小
b2 = np.array([0, 2])      # 正上，垂直
b3 = np.array([-2, 1])     # 左上，夹角大

print(np.dot(a, b1))   # 6    → 正数 = 锐角（< 90°）
print(np.dot(a, b2))   # 0    → 零 = 垂直（恰好 90°）
print(np.dot(a, b3))   # -6   → 负数 = 钝角（> 90°）
```

**判断规则**：

| 点积 | cos(θ) | θ 范围 | 含义 |
|------|--------|--------|------|
| > 0 | > 0 | 0° ~ 90° | 大致同向 |
| = 0 | = 0 | 恰好 90° | 垂直 |
| < 0 | < 0 | 90° ~ 180° | 大致反向 |

### 最重要的几何直觉：投影

> **a 在 b 方向上的投影长度 = `(a·b) / |b|`**

```python
a = np.array([3, 2])       # 任意方向
b = np.array([3, 0])       # x 轴方向
b_hat = b / np.linalg.norm(b)   # x 轴单位向量 = [1, 0]

proj_len = np.dot(a, b_hat)      # = 3×1 + 2×0 = 3
proj_vector = proj_len * b_hat   # = [3, 0]
```

**画在纸上**：a=(3,2) 是一个从原点出发指到 (3,2) 的箭头。把它投影到 x 轴（即 b 的方向），投影点就是 (3,0)。投影长度 = 3 恰等于 a 在 x 轴上的分量。

**这就是投影的全部**：`a·b̂` 算出"a 在 b 指向的方向上走了多远"。

### 3DGS 中怎么用点积？

**场景1：判断高斯在不在相机视野里**

相机的视线方向是 d（单位向量），从相机到高斯的向量是 v。如果 `d·v > 0`，说明高斯在相机前方；如果 `< 0`，在身后（扔掉不渲染）。

**场景2：球谐函数计算**

球谐函数本质是视线方向 d 和球谐基函数 `Y_lm` 的某种"方向相似度"计算，内积形式。

## 3.4 叉积 —— 造出一个垂直于两个向量的新向量

### 定义（仅限 3D 空间）

```python
a = np.array([2, 0, 0])    # x 轴方向
b = np.array([0, 2, 0])    # y 轴方向

c = np.cross(a, b)
print(c)   # [0, 0, 4]    ← z 轴方向，垂直于 xy 平面
```

### 手算公式（用行列式记忆）

```
a × b = det( | i   j   k |
             | a₁  a₂  a₃ |
             | b₁  b₂  b₃ | )

     = i(a₂b₃ - a₃b₂) - j(a₁b₃ - a₃b₁) + k(a₁b₂ - a₂b₁)
```

**一步一步算 a=(2,-1,3), b=(1,4,-2)**：

```
i 分量：a₂b₃ - a₃b₂ = (-1)×(-2) - 3×4 = 2 - 12 = -10
j 分量：a₁b₃ - a₃b₁ = 2×(-2) - 3×1 = -4 - 3 = -7  → 前面有负号 → +7
k 分量：a₁b₂ - a₂b₁ = 2×4 - (-1)×1 = 8 + 1 = 9

a × b = (-10, 7, 9)
```

### 验证：结果垂直于 a 和 b

```python
a = np.array([2, -1, 3])
b = np.array([1, 4, -2])
c = np.cross(a, b)

# 验证垂直性：两个垂直向量的点积 = 0
print(np.dot(c, a))   # -10×2 + 7×(-1) + 9×3 = -20-7+27 = 0 ✓
print(np.dot(c, b))   # -10×1 + 7×4 + 9×(-2) = -10+28-18 = 0 ✓
```

### 几何意义

**方向：右手定则**

右手食指 = a，中指 = b，拇指 = a×b。它永远指向 a 和 b 所在平面**的法向量**方向。

**大小：平行四边形面积**

```
|a × b| = |a|·|b|·sin(θ) = a 和 b 张成的平行四边形面积
```

验证 `a=(2,0,0), b=(0,2,0)`：|a|×|b|×sin(90°) = 2×2×1 = 4，而 `|c| = |(0,0,4)| = 4` ✓

### 关键性质

```python
# 1. 自己叉自己 = 零向量
np.cross(a, a)   # [0, 0, 0]

# 2. 反交换律：a×b = -(b×a)
np.allclose(np.cross(a, b), -np.cross(b, a))   # True
```

### 3DGS 中怎么用叉积？

**计算平面法向量**：建筑场景中，给定墙面上的三个点 p1, p2, p3，墙面的法向量方向 = `(p2-p1) × (p3-p1)`。这对于判断高斯椭球是否应该"躺平"贴在墙面上非常关键。

---

## 3.5 点积 vs 叉积 — 对比总结

```
           点积 a·b                    叉积 a×b
           ────────                   ────────
结果类型    标量（一个数）              向量（有方向有长度）
计算       a₁b₁ + a₂b₂ + a₃b₃         见行列式展开
几何意义   度量"对齐程度"              造法向量 + 算面积
方向信息   cos(θ) → 同向/垂直/反向     垂直于 a 和 b（右手定则）
等于零时   垂直                        平行或一个向量为零
3DGS用途   视野判断、球谐投影           法向量计算
```

# 板块四详解：矩阵——线性变换的语言

板块四是今天最重要的一节。它回答一个问题：**矩阵到底是什么？** 答案是：矩阵不是"数字表格"，它是**空间变形操作**的编码。

---

## 4.1 从你已知的概念出发

你现在是 Python 水平。一个 2×2 矩阵在代码里长这样：

```python
A = np.array([[a, b],
              [c, d]])
```

这是**数据的表示**。但它的**几何含义**是什么？

把矩阵 A 作用于向量 v：

```python
v = np.array([x, y])
result = A @ v       # 结果是一个新向量
```

`A @ v` 的效果：把 v 从原来的位置"搬运"到新位置。**搬运的规则**由矩阵 A 定义。

**A 是一台"空间变形机"**：输入任意向量 → 输出被变形后的向量。

---

## 4.2 矩阵的每一列 = 基向量变换后的位置——这是理解矩阵最重要的直觉

### 什么是基向量？

在 2D 空间中，有两个"标准方向"：

```python
e1 = np.array([1, 0])    # x 轴方向的单位向量
e2 = np.array([0, 1])    # y 轴方向的单位向量
```

任意向量都可以用这两个基向量表示：

```python
v = np.array([3, 2])
# 等价于：v = 3 * e1 + 2 * e2
#            = 3 * [1,0] + 2 * [0,1]
#            = [3, 2]
```

### 关键定理

> **矩阵 A 的每一列，是原来基向量经过 A 变换后的新位置。**

```
A = [[a, b],
     [c, d]]

第一列 (a, c) = e₁=(1,0) 变换后去了哪里
第二列 (b, d) = e₂=(0,1) 变换后去了哪里
```

### 验证：为什么这个定理是对的？

```python
A = np.array([[a, b],
              [c, d]])

# e₁ = [1, 0]，作用 A：
A @ np.array([1, 0])
# = [[a, b],    @ [1,
#    [c, d]]       0]
# = [a×1 + b×0, c×1 + d×0]
# = [a, c]            ← 正好是第一列！

# e₂ = [0, 1]，作用 A：
A @ np.array([0, 1])
# = [a×0 + b×1, c×0 + d×1]
# = [b, d]            ← 正好是第二列！
```

**任何向量 v = v₁e₁ + v₂e₂ 变换后**：

```
A @ v = A @ (v₁e₁ + v₂e₂)
      = v₁·(A@e₁) + v₂·(A@e₂)
      = v₁·(第一列) + v₂·(第二列)
```

**这就是线性变换的全部**：你只需要知道基向量 e₁ 和 e₂ 被搬到了哪里，就能推知**任意向量**被搬到了哪里。

---

## 4.3 六种基本变换——对照 demo_01 理解

demo_01 的 `demo_matrix_as_transform()` 函数展示了六种矩阵对**单位正方形**的影响。单位正方形 `[0,0]→[1,0]→[1,1]→[0,1]` 是观察变换的标准参考物。

### 变换1：单位矩阵——不变

```python
I = np.array([[1, 0],
              [0, 1]])

# 第一列 = (1,0) = e₁ 原位不动
# 第二列 = (0,1) = e₂ 原位不动
```

正方形纹丝不动。就像乘以 1。

### 变换2：水平拉伸 ×2

```python
A = np.array([[2, 0],
              [0, 1]])

# 第一列 = (2,0) → e₁ 被水平拉到原来 2 倍
# 第二列 = (0,1) → e₂ 不变
```

效果：正方形在 x 方向被拉成原来的 2 倍宽度。`det(A) = 2×1 = 2`，面积变为原来的 2 倍。

### 变换3：垂直拉伸 ×2

```python
A = np.array([[1, 0],
              [0, 2]])

# 第一列 = (1,0) → e₁ 不变
# 第二列 = (0,2) → e₂ 被垂直拉到 2 倍
```

效果：正方形在 y 方向变高。`det = 2`。

### 变换4：旋转 90°（逆时针）

```python
theta = np.radians(90)
A = np.array([[0, -1],
              [1,  0]])
# cos90°=0, sin90°=1 代入旋转矩阵公式

# 第一列 = (0, 1)    → e₁ 从指向右变成指向上
# 第二列 = (-1, 0)   → e₂ 从指向上变成指向左
```

效果：整个正方形绕原点转了 90°。`det = 0×0 - (-1)×1 = 1`，旋转不改变面积。

**通用旋转矩阵公式**：
```python
theta = np.radians(angle)
R = np.array([[np.cos(theta), -np.sin(theta)],
              [np.sin(theta),  np.cos(theta)]])
```

### 变换5：水平剪切

```python
A = np.array([[1, 0.5],
              [0, 1  ]])

# 第一列 = (1, 0) → e₁ 不变
# 第二列 = (0.5, 1) → e₂ 向右倾斜！
```

效果：正方形变成平行四边形——顶边向右偏了 0.5。**注意**：`det = 1×1 - 0.5×0 = 1`，剪切不改变面积！

### 变换6：复合变换

```python
A = np.array([[2, 0.5],
              [0.3, 1.5]])

# 同时包含拉伸 + 剪切 + 旋转
```

`det = 2×1.5 - 0.5×0.3 = 3 - 0.15 = 2.85`，面积扩大了 2.85 倍。

**这六种变换要亲自看 demo_01 的图**——红箭头 = 变换后的 e₁，蓝箭头 = 变换后的 e₂。观察基向量的去向，你就能直观理解矩阵在做什么。

## 4.4 矩阵乘法 = 变换的复合

```python
A @ B @ v    # 先做 B 变换，再做 A 变换
             # 等价于：用 (A @ B) 一次性做两个变换
```

**顺序极其重要**：AB ≠ BA。

**例子：先旋转再拉伸 vs 先拉伸再旋转**

```python
# 先在原点旋转 90°，再水平拉伸
R = np.array([[0, -1], [1, 0]])     # 旋转 90°
S = np.array([[2, 0], [0, 1]])     # 水平拉伸 ×2

# 先旋转后拉伸
M1 = S @ R
# M1 @ v = 先旋转(R)，再拉伸(S)

# 先拉伸后旋转
M2 = R @ S
# M2 @ v = 先拉伸(S)，再旋转(R)

# M1 ≠ M2！（试试用代码验证）
```

**对应的 3DGS 理解**：在 OpenGL 中你写过 `gl_Position = projection * view * model * vertex`，矩阵从右到左依次作用——先 model，再 view，再 projection。顺序绝不能反。

## 4.5 行列式 = 面积（2D）或体积（3D）的缩放比

```python
A = np.array([[2, 0],
              [0, 3]])
det = np.linalg.det(A)   # = 6.0
```

**含义**：单位正方形面积=1，经过 A 变换后面积变为 6。

### 几个关键值

```python
# det = 1：面积不变
R = np.array([[0, -1], [1, 0]])
np.linalg.det(R)   # 1.0 —— 旋转不改变面积

# det = 0：空间被"压扁"——不可逆
Z = np.array([[1, 2],
              [1, 2]])
np.linalg.det(Z)   # 0.0 —— 两列线性相关，正方形变成一条线段

# det < 0：发生了翻转（镜像）
M = np.array([[-1, 0],
              [ 0, 1]])
np.linalg.det(M)   # -1.0 —— x 方向翻转
```

### 当 det = 0 时

```python
Z = np.array([[1, 2],
              [1, 2]])
# 两列完全成比例：(1,1)×2 = (2,2)
# 两个基向量变换后指向同一条直线上
# 整个 2D 空间被压成了一条 1D 的线
# 逆矩阵不存在——因为"压扁"是单向的，无法恢复
```

**3DGS 中的意义**：如果协方差矩阵 Σ 的特征值之一趋近 0，高斯椭球从 3D "压扁"成 2D 平面，甚至 1D 线段。这在自适应密度控制中会被检测和修剪。

---

## 4.6 逆矩阵 —— 变换的"撤销"操作

```python
A = np.array([[2, 0],
              [0, 3]])          # 水平×2，垂直×3
A_inv = np.linalg.inv(A)
# [[0.5, 0. ],
#  [0. , 0.333]]

# 验证：先放大再缩小 = 不变
np.allclose(A @ A_inv, np.eye(2))   # True
```

**几何含义**：A⁻¹ 把被 A 拉伸后的空间恢复原状。

**存在条件**：`det(A) ≠ 0`。

**与旋转矩阵的关键关系**：

```python
theta = np.radians(45)
R = np.array([[np.cos(theta), -np.sin(theta)],
              [np.sin(theta),  np.cos(theta)]])

# 旋转矩阵满足：转置 = 逆矩阵
np.allclose(R.T, np.linalg.inv(R))   # True

# 因此：旋转矩阵求逆只需 .T，不需要昂贵的 inv 调用
```

**3DGS 中的用途**：相机旋转矩阵的逆（`R_camᵀ`）用于把点从相机坐标转回世界坐标——不需要调 `inv`，直接 `.T`。

## 4.7 转置矩阵 —— 行变列，列变行

### 基本操作

```python
A = np.array([[1, 2, 3],
              [4, 5, 6]])          # (2, 3)

A.T                                # (3, 2)
# [[1, 4],
#  [2, 5],
#  [3, 6]]
```

`(Aᵀ)ᵢⱼ = Aⱼᵢ`：第 i 行第 j 列 → 第 j 行第 i 列。

### 三个必须记住的性质

**性质1：`(AB)ᵀ = BᵀAᵀ`——顺序反转！**

```python
A = np.array([[1, 2], [3, 4]])
B = np.array([[5, 6], [7, 8]])

print(np.allclose((A @ B).T, B.T @ A.T))   # True
print(np.allclose((A @ B).T, A.T @ B.T))   # False —— 顺序错了！
```

这个性质在 3DGS 中反复出现：当你推导 `(R·S·Sᵀ·Rᵀ)ᵀ = R·S·Sᵀ·Rᵀ` 时（协方差矩阵的对称性），本质就是反复用这条。

**性质2：对称矩阵满足 `Aᵀ = A`**

```python
Sigma = np.array([[3, 1],
                  [1, 3]])
print(np.allclose(Sigma.T, Sigma))   # True
```

协方差矩阵永远是对称的（因为 `Cov(X,Y) = Cov(Y,X)`）。

**性质3：旋转矩阵满足 `Rᵀ = R⁻¹`**

```python
# 这就是"正交矩阵"的定义
# 旋转不改变向量长度 → R 的每一列都是单位向量且互相垂直
# → Rᵀ @ R = I → Rᵀ = R⁻¹
```

## 4.8 板块四总结：一个核心公式

整个 3DGS 的线性代数基础可以浓缩为一条变换链：

```
向量 v ──[ R @ S ]──→ 旋转+缩放后的向量
             │
     R = 旋转矩阵（四元数转换而来）
     S = 对角缩放矩阵（diag(s₁, s₂, s₃)）
             │
协方差 Σ = (R·S) @ (R·S)ᵀ
         = R @ S @ Sᵀ @ Rᵀ
```

**这条链上的每一步都对应你今天学的一个 numpy 操作**：

```python
s = np.array([2.0, 0.5, 1.0])                   # 缩放向量
S = np.diag(s)                                    # 缩放矩阵（板块二 diag）
theta = np.radians(45)                            # 角度→弧度（板块二 radians）
R = np.array([[np.cos(theta), -np.sin(theta), 0], # 旋转矩阵
              [np.sin(theta),  np.cos(theta), 0],
              [0,              0,             1]])
M = R @ S                                         # 复合变换（板块一 @）
Sigma = M @ M.T                                   # 协方差矩阵（板块一 @ + .T）

eigenvalues, Q = np.linalg.eigh(Sigma)             # 验证正定性（板块一 eigh）
print(eigenvalues)   # 应全为正 → [4. 0.25 1.] 对应 (sx², sy², sz²)
```

## 5.2 第一部分：点积的几何意义 —— `demo_dot_product()`

脚本会弹出两张图。

### 左图：为什么 a·b = |a||b|cosθ

图中一根红色长箭头 `a = (3, 0)` 指向正右方，三根不同颜色的箭头 `b` 与 a 的夹角各不相同。

蓝色 b 与 a 夹角最小 → `a·b` 的值最大（正向对齐）。
橙色 b 与 a 垂直 → `a·b = 0`（cos 90° = 0，对齐程度为零）。
绿色 b 方向在两者之间 → `a·b` 的值居中。

每根箭头旁标注了实际算出的点积值。这是"点积 = 对齐程度"的视觉证据。

### 右图：投影

`a = (3, 2)`（红色箭头，斜的），`b = (3, 0)`（蓝色箭头，沿 x 轴）。

绿色箭头 = a 在 b 方向上的投影。

虚线从 a 的顶端垂直落到 b 所在直线上，绿色箭头的末端恰是垂足。绿色箭头的长度 = `a·b̂`。

**这图的核心信息**：点积不是凭空定义的——它就是在度量"一个向量在另一个向量方向上走了多远"。

## 5.4 第三部分：矩阵 = 线性变换 —— `demo_matrix_as_transform()`

这是今天最关键的可视化——一张图上有六幅子图。

### 怎么看这六幅图

每幅图的构成：
- 灰色虚线正方形 = **原始单位正方形** `[0,0]→[1,0]→[1,1]→[0,1]`
- 彩色填充区域 = 经过矩阵变换后的形状
- 红色箭头 = 变换后的 e₁（第一列）
- 蓝色箭头 = 变换后的 e₂（第二列）
- 左上角标注了 `det(M)`

### 逐幅解读

**子图1：单位矩阵 `[[1,0],[0,1]]`**
红箭头指向 (1,0)，蓝箭头指向 (0,1)——基向量没动。正方形纹丝不动。`det = 1`。

**子图2：水平拉伸 `[[2,0],[0,1]]`**
红箭头变长 (2,0)，蓝箭头不变 (0,1)。正方形在 x 方向被拉成两倍宽度的矩形。`det = 2`——面积翻倍。

**子图3：垂直拉伸 `[[1,0],[0,2]]`**
红箭头不变，蓝箭头变长 (0,2)。正方形在 y 方向拉高。`det = 2`。

**子图4：旋转 90° `[[0,-1],[1,0]]`**
红箭头指向 (0,1)（原指右，现指上），蓝箭头指向 (-1,0)（原指上，现指左）。整个正方形绕原点转了 90°。但 `det = 1`——旋转不改变面积。

**子图5：水平剪切 `[[1,0.5],[0,1]]`**
红箭头没动 (1,0)，但蓝箭头偏了 (0.5,1)。正方形变成了平行四边形。`det = 1`——剪切不改变面积！

**子图6：复合变换 `[[2,0.5],[0.3,1.5]]`**
两个基向量都偏离了原始方向。正方形变成不规则四边形。`det ≈ 2.85`——面积扩张。

### 看这六幅图时你需要想的核心问题

"已知基向量变换后的位置（红蓝箭头），我能还原这个变换吗？"

**能**。因为任意点 `(x, y) = xe₁ + ye₂` 变换后 = `x·(红箭头) + y·(蓝箭头)`。
